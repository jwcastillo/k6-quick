# k6-quick

Load test any URL with a single command — no script authoring required.

[k6](https://k6.io) is script-first: even the simplest smoke test needs a
JavaScript file. `k6-quick` generates that script for you from CLI flags and
pipes it straight to `k6 run` — no temp files, nothing to clean up. When you
outgrow the one-liner, `--print-script` hands you the exact script it runs so
you can save and evolve it.

## Install

```bash
go install github.com/jwcastillo/k6-quick@latest
```

Requires Go 1.25+ and a `k6` binary on your `PATH`
([install k6](https://grafana.com/docs/k6/latest/set-up/install-k6/)).

## Usage

```bash
# 50 virtual users for 30 seconds
k6-quick https://api.example.com/health -c 50 -d 30s

# fixed iteration count instead of duration
k6-quick https://api.example.com/health -c 10 -n 1000

# open-model: 200 requests/second arrival rate
k6-quick https://api.example.com/health --rps 200 -d 1m

# POST with headers, body, and a pass/fail threshold
k6-quick https://api.example.com/orders \
  -m POST \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -b '{"sku":"A-1"}' \
  --threshold "http_req_duration:p(95)<500"
```

k6's own output streams through untouched, and k6's exit code is propagated
verbatim — so threshold failures keep k6's standard exit code `99` and work
with any tooling that already understands k6.

## Flags

| Flag | Default | Maps to |
|------|---------|---------|
| `-c, --vus` | `10` | `options.vus` |
| `-d, --duration` | `30s` | `options.duration` |
| `-n, --iterations` | — | `options.iterations` (replaces duration) |
| `--rps N` | — | `constant-arrival-rate` scenario (rate `N`/s, `preAllocatedVUs: c`, `maxVUs: 2×c`) |
| `-m, --method` | `GET` | request method |
| `-H, --header` | — | request header, repeatable (`"Key: Value"`) |
| `-b, --body` | — | request body |
| `--threshold` | — | k6 threshold expression, repeatable (`"metric:expr"`) |
| `--print-script` | — | print the generated script and exit (k6 not required) |

## Graduate to a Real Script

Everything is baked into the generated script's `options` — what you see with
`--print-script` is exactly what runs:

```bash
k6-quick https://api.example.com/health -c 50 -d 30s \
  --threshold "http_req_duration:p(95)<500" --print-script
```

```javascript
// generated by k6-quick — save with --print-script to keep this script
import http from "k6/http";

export const options = {
  vus: 50,
  duration: "30s",
  thresholds: {
    "http_req_duration": ["p(95)<500"],
  },
};

export default function () {
  http.request("GET", "https://api.example.com/health", null, { headers: {} });
}
```

Redirect it to a file and you have a normal, versionable k6 script:

```bash
k6-quick https://api.example.com/health -c 50 -d 30s --print-script > load-test.js
k6 run load-test.js
```

## Safety Notes

- The script reaches k6 via stdin (`k6 run -`) — zero temp files.
- Every user-supplied value (URL, headers, body, thresholds) is JSON-encoded
  into the script, so quotes, backticks, and newlines cannot break out of the
  generated JavaScript.
- URLs must be `http://` or `https://`; malformed headers and methods are
  rejected with clear errors before k6 is ever invoked.

## Running the Tests

```bash
go test -race ./...
```

The suite covers golden-file script generation (every flag combination,
including special characters in URLs/headers/bodies), input validation, the
`--print-script` path, the missing-k6 error message, and — when a `k6` binary
is available — validates every generated script with `k6 inspect`. Golden
files regenerate with `go test -run TestGenerate -update ./...`.
